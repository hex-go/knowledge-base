# 切片扩容机制

## 标准知识

Go 的切片在 `append` 触发扩容时，会根据当前容量选择不同的增长策略：

- **容量 < 256**（Go 1.18+）：扩容为原来的 **2 倍**
- **容量 >= 256**：增长约 **25%**（渐进式，大切片趋近于 1.25 倍）

扩容过程：
1. 计算新容量
2. 根据新容量分配新的底层数组（大小按内存对齐向上取整）
3. 将老数据拷贝到新数组
4. 返回指向新数组的切片

## 我的理解

可以把切片想象成一个"窗台花盆"——开始时用一个小花盆（底层数组），花长到装不下了就换大花盆。Go 每次换盆的策略其实很聪明：

1. 小切片（< 256 个元素）：直接空间翻倍，因为你很小，翻倍代价不高
2. 大切片（>= 256）：保守增长 25%，避免浪费太多内存

关键理解：扩容后旧数组会被 GC 回收，但如果有其他切片也指向旧数组，它们看到的还是旧数据。这就是为什么 `append` 后一定要使用返回值。
